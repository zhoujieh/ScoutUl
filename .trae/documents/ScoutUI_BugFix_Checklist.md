# ScoutUI 插件问题修复 - 验证清单

## 验证步骤

### 步骤1: 加载插件
- [ ] 打开 Chrome 扩展管理页面 (chrome://extensions/)
- [ ] 开启"开发者模式"
- [ ] 点击"加载已解压的扩展程序"
- [ ] 选择 ScoutUI 文件夹
- [ ] 确认插件显示在扩展列表中

### 步骤2: 检查初始化
- [ ] 打开测试页面 (http://localhost:8080/test.html)
- [ ] 打开 Chrome 开发者工具 (F12)
- [ ] 切换到"控制台"选项卡
- [ ] 刷新测试页面
- [ ] 检查控制台是否显示初始化日志:
  - [ ] "ScoutUI: 开始初始化..."
  - [ ] "ScoutUI: 覆盖层创建成功"
  - [ ] "ScoutUI: 信息面板创建成功"
  - [ ] "ScoutUI: 鼠标移动事件监听器已添加"
  - [ ] "ScoutUI: 消息监听器已添加"
  - [ ] "ScoutUI: 键盘事件监听器已添加"
  - [ ] "ScoutUI: 初始化完成"

### 步骤3: 测试 popup 界面
- [ ] 点击 Chrome 浏览器右上角的插件图标
- [ ] 确认 popup 界面正常显示
- [ ] 确认"启用走查模式"开关可见

### 步骤4: 测试走查模式
- [ ] 点击"启用走查模式"开关
- [ ] 将鼠标悬停在测试页面的任意元素上
- [ ] 观察是否显示元素高亮边框
- [ ] 观察是否显示信息面板

### 步骤5: 测试快捷键
- [ ] 关闭走查模式（点击开关）
- [ ] 按下 Alt+Shift+S
- [ ] 确认走查模式已开启
- [ ] 再次按下 Alt+Shift+S
- [ ] 确认走查模式已关闭

### 步骤6: 测试状态持久化
- [ ] 开启走查模式
- [ ] 刷新测试页面
- [ ] 确认走查模式仍然开启

## 错误排查

如果发现问题，检查以下内容：
1. 控制台是否有错误信息
2. manifest.json 配置是否正确
3. content.js 是否正确加载
4. CSS 文件是否正确引用
