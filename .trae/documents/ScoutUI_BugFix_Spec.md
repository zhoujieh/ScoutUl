# ScoutUI 插件问题修复 - 详细规格说明书

## 1. 项目概述

- **项目名称**: ScoutUI - Chrome UI自动走查工具
- **项目类型**: Chrome 浏览器扩展程序
- **核心功能**: 当用户将鼠标悬停在网页元素上时，自动检测并显示该元素的布局方式、内外边距数值、尺寸信息、定位方式、字体样式、颜色值以及其他关键UI属性
- **目标用户**: 前端开发者、UI设计师、测试工程师

## 2. 当前问题描述

插件加载后无法正常工作，具体表现：
1. 开启走查模式后无任何反应
2. 鼠标悬停在元素上不显示高亮和信息面板
3. 快捷键 Alt+Shift+S 无法切换走查模式

## 3. 问题根因分析

通过代码审查和错误日志分析，发现以下可能的问题：
1. `content.js` 初始化逻辑可能存在问题
2. 消息传递机制可能失败
3. CSS 样式可能未正确加载
4. 存储权限可能未正确配置

## 4. 修复方案规格

### 4.1 功能规格

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 元素悬停检测 | 鼠标悬停在元素上时自动检测并高亮 | P0 |
| 信息面板显示 | 显示元素详细信息（布局、尺寸、颜色等） | P0 |
| 快捷键支持 | Alt+Shift+S 切换走查模式 | P1 |
| 状态持久化 | 保存走查模式状态 | P1 |

### 4.2 技术规格

| 项目 | 规格 |
|------|------|
| manifest_version | 3 |
| 权限 | activeTab, storage |
| 内容脚本 | content.js + overlay.css + panel.css |
| 后台服务 | background.js |

### 4.3 验收标准

- [ ] 插件能成功加载到 Chrome 扩展管理页面
- [ ] 点击插件图标能打开 popup 界面
- [ ] 开启走查模式后，鼠标悬停能显示元素高亮
- [ ] 鼠标悬停时能显示信息面板，包含元素的布局、尺寸、颜色等信息
- [ ] 快捷键 Alt+Shift+S 能切换走查模式
- [ ] 刷新页面后，走查模式状态能正确恢复
